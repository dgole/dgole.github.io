NLP Preprocessing¶

I'm following the guide at the following link pretty closely to do LDA and topic modeling on the dataset.
https://www.analyticsvidhya.com/blog/2018/10/mining-online-reviews-topic-modeling-lda/
The basic procedure is:

  1. Split each review into a list of words
  2. "Lematize" each word: reduce it to it's root word
  3. Remove stopwords from a database of stopwords. These are words that don't really add information (and, the, etc.)
In [9]:
def lemmatize_stemming(text):
    return stemmer.stem(WordNetLemmatizer().lemmatize(text, pos='v'))
# Tokenize and lemmatize
def preprocess(text, exclude_words):
    result=[]
    for token in gensim.utils.simple_preprocess(text) :
        if token not in gensim.parsing.preprocessing.STOPWORDS and len(token) > 3:
            res = lemmatize_stemming(token)
            if res not in exclude_words:
                result.append(res)
    return result

Topic Modeling¶

  1. Use gensim to make a dictionary of words --> integer mapping over all the reviews
  2. Create a word count frequency matrix for each individual review
In [11]:
class LDA_wrapper():
    def __init__(self, df, col, n_topics):
        self.n_topics        = n_topics
        self.corpus_all      = df[col].as_matrix()
        self.dictionary      = gensim.corpora.Dictionary(self.corpus_all)
        self.doc_term_matrix = [self.dictionary.doc2bow(rev) for rev in df[col]]
        LDA                  = gensim.models.ldamodel.LdaModel
        self.lda_model       = LDA(corpus=self.doc_term_matrix,
                                   id2word = self.dictionary,
                                   num_topics=self.n_topics, 
                                   random_state=0, 
                                   )

Remove common and unhelpful words¶

Words like stay, hotel, room, night are very common and don't add any information.
Words like good, great, nice aren't helpful in determining topics, and we already have overall happiness to determine if the reviewer liked the hotel.
Let's exclude all of these.

Compute a simple metric:¶

For each hotel, calculate:
happiness by topic = total amount of topic in happy reviews / total amount of topic in all reviews
unhappiness by topic = total amount of topic in unhappy reviews / total amount of topic in all reviews
topic score = happiness in the topic - unhappiness in that topic adjusted score = topic score for given hotel - average topic score across all hotels

def make_topic_happiness_plot(df_diffs, n_topics=3):
    fig = plt.figure(figsize=(6,4), dpi=150)
    x0  = (n_topics+3)*np.arange(len(df_diffs))
    for i in range(n_topics):
        plt.bar(x0+i+0.5, df_diffs['diff_topic_'+str(i)].values, label=topic_names[i])
    #for i in range(len(df)):
        #plt.axvline(x=x0[i]+n_topics+0.5, color='k')
    plt.xticks(x0+float(n_topics)/2.0, df_diffs.index , rotation=0)
    #plt.ylabel('Freq in Pos Revs - Freq in Neg Revs')
    plt.xlabel('Hotel ID')
    plt.axhline(y=0, color='k')
    plt.title('Hotel Performance by Topic')
    plt.ylim(-1.5, 1.5)
    plt.text(6, 1.25,  "reviews on this topic were happier than average", fontsize=11)
    plt.text(6, -1.35, "reviews on this topic were less happy than average", fontsize=11)
    plt.legend(loc=(1.01, 0.7))
    return plt
